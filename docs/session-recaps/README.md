# Session Recaps

This directory contains chronological recaps of Claude Code work sessions
on the Pecunia project. Each recap is generated at the end of a session
via the `/recap` slash command.

The recaps complement other documentation:

- **`JOURNAL.md`** (at repository root) provides high-level entries:
  what was done, what was learned, what's next. One short entry per
  session.
- **Session recaps** (here) provide the detailed account of each
  session: files modified, decisions made, concepts discussed, issues
  encountered.

## Naming convention

Files are named `YYYY-MM-DD-session-NN.md` and organized in monthly
folders (`YYYY-MM/`), where `NN` is a zero-padded session number that
increments within the month.

## Purpose

- Track detailed progress for personal reference.
- Preserve context for resuming work after breaks.
- Document the construction history of the project.
- Provide source material for technical discussions in interviews.
