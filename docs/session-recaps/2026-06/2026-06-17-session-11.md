## Key decisions made
- **M3 palette: Azure/Blue.** Aligns with the banking/finance visual code
  (trust); reversible toward a custom brand theme later.
- **API namespaced under `/api` now** (vs keeping `/me` at root). Single
  prefix to proxy, clean separation from SPA routes and Spring Security
  endpoints; cost paid while only one endpoint exists.
- **Generated TS client gitignored + regenerated** (vs committed). Mirrors
  the backend `target/` output; the contract stays the single source of
  truth. Output in `src/generated/api` to mark it clearly as generated.
- **`changeOrigin: false`** on the proxy so Spring sees the original
  `localhost:4200` Host header and derives the OAuth2 redirect-uri on that
  origin — required for the BFF session cookie to land where the SPA can
  use it.
- **Generator engine pinned to 7.22.0** (CLI wraps the same Java generator
  as the backend Maven plugin) for identical contract interpretation.

## Concepts explained / learned
- **`mat.theme()` M3 API**: emits `--mat-sys-*` CSS variables directly in
  `styles.scss`; the legacy `mat-typography` / `mat-app-background` body
  classes are no longer needed.
- **BFF same-origin requirement**: the dev proxy exists so the SPA and
  backend share the `:4200` origin, a precondition for the session cookie
  to work end to end.
- **Generated client basePath resolution**: `BaseService` falls back to
  `http://localhost` only when no string basePath is provided; the spec has
  no `servers`, so `provideApi('')` must be used to force relative `/api/me`
  URLs that flow through the proxy.
- **npm pre-hooks** (`prestart`, `prebuild`) guarantee the gitignored client
  is generated before serve/build.
