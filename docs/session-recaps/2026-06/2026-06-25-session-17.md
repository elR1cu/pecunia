## Concepts explained / learned
- **ESLint vs Prettier**: Prettier governs how code *looks* (formatting, never
  changes meaning); ESLint governs what the code *does* (bugs, types, idioms,
  a11y) by reasoning over the AST. Complementary, hence eslint-config-prettier.
- **Maven BOM import scope limitation**: `<scope>import</scope>` pulls only the
  BOM's `dependencyManagement`, not its `pluginManagement` or properties — so
  the CycloneDX plugin would need an explicit version despite being in the BOM.
- **SBOM value without exposure**: an SBOM's worth is in controlled distribution
  (CI artifact, release attachment, Dependency-Track for continuous CVE
  monitoring, incident response), not in serving `/actuator/sbom` to the network.
- **npm strict-peer ERESOLVE**: npm v11 cannot do an in-place lockstep bump of
  Angular framework packages (exact inter-peer deps); `--legacy-peer-deps` on a
  genuinely-consistent set produces a valid lock that `npm ci` then accepts.

## Issues encountered & resolved
- **Misdiagnosis (self-corrected)**: first blamed `openapi-generator-cli@2.39.0`
  for a frontend build break; the real cause was the local Node 22.9.0 vs the
  project's pinned Node 24.16.0 (`require(ESM)` needs Node ≥ 22.12). Builds are
  green on Node 24.
- **Lockfile over-churn**: deleting `package-lock.json` re-floated 74 unrelated
  transitives. Reverted to baseline and used a surgical `npm install <angular>
  --legacy-peer-deps` keeping the lock for everything else (diff: Angular
  closure + 2 trivial).
- **ESLint on generated code**: 38 initial errors were all in `src/generated`;
  fixed by ignoring it. Remaining 5 were legitimate test-double patterns, fixed
  via a spec-file rule relaxation, not by editing tests.
