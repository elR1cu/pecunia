## What was done

- **Validated end to end in the browser**: login → dashboard → logout button →

## Key decisions made
- **Logout via Option B** (AJAX POST returning a JSON `logoutUrl`, then
  `window.location.href`) over Option A (full-page form POST with manual
  `_csrf`) or Option C (local-session-only logout). B is CSRF-native via the
  Angular header and achieves true RP-initiated logout (kills the Keycloak SSO
  session), unlike C.
- **Reuse the existing handler via a custom `RedirectStrategy`** rather than
  reimplementing the OIDC end-session URL: only the transport changes (JSON vs
  302), all URL-building logic is inherited.
- **i18n = ngx-translate v18** (ADR-0025): the app is entirely behind auth
  (build-time `@angular/localize`'s SEO/initial-load advantage is moot) and is
  zoneless/signals-first (v18 is signal-native). Transloco is the designated
  fallback if ngx-translate's maintenance revival stalls.
- **Adopt the i18n mechanism now, translate incrementally** rather than
  retrofitting later (retrofit cost: per-template string sweep).
- **No `Co-Authored-By` trailer on commits**, to match the repo's established
  convention (no prior commit uses it; the user authors the code).

## Concepts explained / learned
- **CSRF default alignment**: Angular's `provideHttpClient()` enables XSRF
  protection by default; with Spring `spa()` the default cookie/header names
  match, so no explicit configuration is needed. The header only attaches to
  mutating requests on relative URLs (so `/logout` through the proxy works).
- **BFF logout asymmetry**: an XHR cannot follow a 302 to Keycloak (cross-origin
  navigation), hence the JSON-URL-then-navigate split — AJAX for the
  CSRF-protected session kill, full navigation for the IdP logout.
- **DevTools cannot show the `/logout` response body**: response bodies are
  fetched lazily from the document's resource store, which the immediate
  navigation destroys → "No resource with given identifier found", even with
  Preserve log. A breakpoint on the navigation line pauses before the document
  is destroyed and makes the body inspectable.
- **`@Service()` is a real Angular 22 decorator** — ergonomic shorthand for
  `@Injectable({ providedIn: 'root' })`. Saved as memory; see Issues below.
- **2026 i18n landscape**: `@angular/localize` is still build-time with no
  runtime switch; ngx-translate v18 (June 2026) is rebuilt on Signals;
  Transloco remains the mature runtime alternative.
- **`/logout` is out of the OpenAPI contract** because it is served by a Spring
  Security filter, not a controller — the contract describes the business API
  the SPA consumes via the generated client, not auth infrastructure.

## Issues encountered & resolved
- **Wrongly flagged `@Service()` as a non-existent decorator / compile error**,

  confirmed `@Service` is valid in Angular 22. Self-corrected and saved memory
  `project-angular-22-service-decorator`. Lesson recorded: verify framework API
  claims on this bleeding-edge stack before asserting.
- **Generated `auth-service.spec.ts` stub would have broken the suite**
  (`NullInjectorError: No provider for HttpClient`) — resolved by rewriting it
  with `provideHttpClient()` + `provideHttpClientTesting()` (Option B).
- **Unused `MatButtonModule` import** left after switching to the standalone
  `MatButton` directive — Prettier does not remove unused imports; removed by hand.
