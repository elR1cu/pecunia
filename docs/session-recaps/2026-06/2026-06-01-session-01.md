## Key decisions made

- **Single-module Maven (Option A)**: consistent with `docs/architecture.md:144`, compatible with Spring Modulith (roadmap), ArchUnit (ADR-0016) enforces architectural rules at test-time.
- **`spring-boot-dependencies` BOM instead of `spring-boot-starter-parent`**: no forced inheritance, explicit plugin control, flexibility for a future monorepo parent.
- **Activate only the `build` contributor now**: `git` contributor deferred to Block 8 (deployment), `java`/`os` reserved for internal environments behind auth.
- **Delegate IntelliJ build to Maven**: the `build-info` goal is Maven-bound; delegating the IDE build to Maven ensures `build-info.properties` is always generated regardless of the launcher.

## Concepts explained / learned

- **`spring-boot-dependencies` BOM vs `spring-boot-starter-parent`**: the parent provides implicit configurations (resource filtering, plugin defaults) that the BOM does not, but the BOM gives more control and avoids forced inheritance.
- **Actuator `InfoContributor`s**: the five contributors (`build`, `git`, `env`, `java`, `os`), their data sources, production leak risks, and default-disabled behaviour since Spring Boot 2.6.
- **`build-info` goal and Maven dependency**: `META-INF/build-info.properties` is generated at the `generate-resources` phase — absent when IntelliJ compiles internally.
- **Graceful shutdown** (`server.shutdown: graceful`): Spring waits for in-flight requests to complete before stopping; essential for zero-downtime rolling deploys on Kubernetes.
- **`show-details: when-authorized`**: load balancer probes see `{"status":"UP"}`; authenticated users see full details (DB, Redis, disk).
