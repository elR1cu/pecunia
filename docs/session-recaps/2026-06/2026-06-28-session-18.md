## Key decisions made
- **Land Block 1 via a local fast-forward push, not the GitHub merge button.**
  `main` requires signed commits + linear history + enforce_admins, and GitHub's
  "Rebase and merge" cannot sign the commits it replays. Since the 77 local
  commits were already signed, a `git merge --ff-only` + `git push` preserved
  them verbatim. Required a one-time relax of branch protection (disable
  `enforce_admins` for the push, re-enable immediately — window ~1s).
- **New standing merge policy: small thematic PRs + squash-merge everywhere.**
  Retired the earlier "rebase-merge structural branches" idea because it needs an
  admin manip each time. Block 1 was the one-time exception to preserve its
  founding history.
- **Skip SonarCloud on Dependabot PRs rather than duplicate the tokens.** GitHub
  keeps a separate Dependabot secret store, so `SONAR_TOKEN_API`/`_FRONTEND`
  (defined under Actions) are not exposed to Dependabot runs. Guarding the Sonar
  steps with `github.actor != 'dependabot[bot]'` keeps the tokens in one place;
  build/test + Trivy still gate those PRs. Safe because `main` has no named
  required status check, so a skipped step does not block merge.

## Concepts explained / learned
- **GitHub cannot sign "Rebase and merge" commits.** It signs the single commit
  produced by Squash or Create-a-merge-commit (with its own GPG key), but not the
  commits replayed by a rebase — hence "Base branch requires signed commits.
  Rebase merges cannot be automatically signed by GitHub."
- **"Require signed commits" validates the PR's branch commits, not just the
  merge result.** An unsigned branch commit blocks the merge even when squashing
  ("Commits must have verified signatures") — squash signing the *result* does
  not exempt the PR's own commits. Correction of an earlier wrong assumption made
  this session.
- **Dependabot runs use a separate, restricted context**: read-only
  `GITHUB_TOKEN` and only Dependabot-store secrets, not Actions secrets.
- **Dependabot secret store is separate by product** (Actions / Dependabot /
  Codespaces never share), including at org level — there is no shared store.
- **`@dependabot rebase` is a PR-comment command**, run server-side by the bot —
  not a UI button and not a local `git rebase` (which would orphan the PR from
  Dependabot). Dependabot also auto-rebases open PRs when the base advances.
